## Health endpointy

| Endpoint | Účel |
| --- | --- |
| `GET /api/health` | Liveness — API běží |
| `GET /api/ready` | Readiness — DB dostupná |

Docker healthcheck API používá `/api/ready`. Frontend kontroluje nginx na `/`.

```bash
curl -fsS https://ticketmaster.cermofi.cz/api/health
curl -fsS https://ticketmaster.cermofi.cz/api/ready
docker compose ps
```

## Nasazení na VPS

Typický postup (klon v `/home/new-ticketmaster`):

```bash
cd /home/new-ticketmaster
git pull origin main
docker compose up -d --build
docker compose exec api ticketmaster-cli db migrate
```

Ověření:

```bash
curl -fsS http://127.0.0.1:3006/api/ready
curl -fsS http://127.0.0.1:3006/docs/ | head
./scripts/post-deploy-smoke.sh
```

Přístup na server: uživatel `cermofi`, pak `su -` pro root operace (bez sudo).

## Post-deploy smoke check

Smoke check je **read-only** — nevytváří produkční data.

Defaultně volá veřejné GET: `/api/health`, `/api/ready`, `/api/meta`.

```bash
./scripts/post-deploy-smoke.sh
# nebo:
docker compose exec api ticketmaster-cli smoke check
```

Volitelně autentizované kontroly (login + `/api/auth/me` + export s `limit=1`):

```bash
SMOKE_ALLOW_AUTH=1 SMOKE_CHECK_EMAIL=... SMOKE_CHECK_PASSWORD=... ./scripts/post-deploy-smoke.sh
```

### Audit a smoke

Všechny smoke požadavky posílají hlavičku **`x-ticketmaster-smoke: 1`**. API middleware potlačí zápis audit logu — smoke ** neznečišťuje** audit.

GitHub Actions: `.github/workflows/post-deploy-smoke.yml`.

## Dokumentace na serveru

- Docs: **`/docs/`** (Fumadocs, služba `docs`)
- API reference: **`/docs/api`** (Scalar)
- OpenAPI: **`/api/openapi.json`**

V produkci jsou `/docs/*` a `/api/openapi.json` dostupné **jen z interní sítě** (loopback + RFC1918). Veřejně zůstávají UI, `/api/health`, `/api/ready` a běžné API pro aplikaci.

## Logy

API loguje každý request (method, path, status, duration, `X-Request-ID`).

```bash
docker compose logs -f api frontend docs
```

Volume `api_logs` → `/app/logs` (pokud aplikace zapisuje do souboru).

## E-mail

- Dev: Mailpit.
- `MAIL_SUPPRESS_SEND=true` — fronta bez odeslání.
- Retry: `ticketmaster-cli notifications retry-failed`.

## Rate limit (provoz)

Reset po false positive:

```bash
docker compose exec api ticketmaster-cli rate-limit list
docker compose exec api ticketmaster-cli rate-limit reset --scope login --ip <IP> --identifier user@example.com
```

Při `WEB_CONCURRENCY>1`: restart API nebo reset pro každý worker.

Detail: [Autentizace](/docs/autentizace).

## Bezpečnost (minimum)

- Změňte `APP_SECRET` a DB hesla mimo dev.
- `CORS_ORIGINS` a `TRUSTED_HOSTS` na reálné domény.
- API přidává security hlavičky (`X-Content-Type-Options`, `X-Frame-Options`, …).

## Rollback

```bash
git checkout <predchozi-commit>
docker compose up -d --build
docker compose exec api ticketmaster-cli db migrate
curl -fsS https://ticketmaster.cermofi.cz/api/ready
```

Migrace jsou dopředné — rollback DB ze zálohy.
