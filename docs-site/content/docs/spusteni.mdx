## Požadavky

- Docker a Docker Compose
- Volitelně: Node.js 20+ pro lokální vývoj docs (`docs-site/`)

## 1. Spuštění aplikace

V kořeni repozitáře:

```bash
cp .env.example .env
docker compose up -d --build
```

API při startu automaticky spustí migrace (`ticketmaster-cli db migrate` v Dockerfile CMD).

## 2. Ověření

| Co | URL / příkaz |
| --- | --- |
| Web UI | http://localhost:3006 |
| API health | http://localhost:3006/api/health |
| API ready (DB) | http://localhost:3006/api/ready |
| Docs | http://localhost:3006/docs/ |

```bash
curl -fsS http://localhost:3006/api/health
curl -fsS http://localhost:3006/api/ready
docker compose ps
```

## 3. Demo data (volitelné)

```bash
docker compose exec api ticketmaster-cli db seed-dev
```

Vytvoří interní uživatele, partnera Acme, klienta a ukázkové tickety. Heslo partnerů odpovídá `TICKETMASTER_DEV_PASSWORD` (výchozí `ChangeMe123!`).

Seed je povolen, pokud `APP_BASE_URL` obsahuje `localhost`, nebo pokud nastavíte `ALLOW_SEED_DEV=true`.

## 4. Přihlášení

| Typ uživatele | Jak se přihlásit |
| --- | --- |
| Partner | E-mail + heslo → `POST /api/auth/login` nebo formulář v UI |
| Interní (dev) | E-mail bez hesla → `POST /api/auth/dev-sso` nebo dev SSO v UI |
| Pozvaný uživatel | Odkaz z invite → aktivace hesla → `POST /api/auth/activate` |

Detailní popis: [Autentizace](/docs/autentizace).

## Zastavení

```bash
docker compose down          # data v DB zůstanou
docker compose down -v       # smaže volume postgres_data
```

## Lokální vývoj bez Dockeru (volitelné)

**Backend** — vyžaduje běžící PostgreSQL a `.env`:

```bash
cd backend
pip install -e ".[test]"
ticketmaster-cli db migrate
uvicorn ticketmaster.api.main:app --reload --port 8000
```

**Frontend** — proxy `/api` na `localhost:8000` (viz `frontend/vite.config.js`):

```bash
cd frontend
npm install
npm start    # port 3000
```

**Docs** (samostatně):

```bash
cd docs-site
npm install
npm run dev  # http://localhost:3007/docs
```

## Post-deploy smoke check

Read-only kontrola po nasazení — **nevytváří data** a **nezapisuje audit** (hlavička `x-ticketmaster-smoke: 1`):

```bash
./scripts/post-deploy-smoke.sh
# nebo:
docker compose exec api ticketmaster-cli smoke check
```

Volitelně autentizované kontroly (read-only):

```bash
SMOKE_ALLOW_AUTH=1 SMOKE_CHECK_EMAIL=... SMOKE_CHECK_PASSWORD=... ./scripts/post-deploy-smoke.sh
```

Více: [Provoz](/docs/provoz).
