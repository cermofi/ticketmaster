## Backend (pytest)

```bash
cd backend
pip install -e ".[test]"
pytest
```

V běžícím API kontejneru:

```bash
docker compose exec api python -m pytest -q
```

### Co se testuje (výběr)

| Soubor / oblast | Zaměření |
| --- | --- |
| `test_business_rules.py` | RBAC, workflow, partner izolace, export |
| `test_auth_partner_sign_in.py` | Sign in as partner |
| `test_auth_back_to_admin.py` | Návrat admina |
| `test_audit_api.py` | Audit API a filtry |
| `test_audit_suppression.py` | Smoke header nezapisuje audit |
| `test_gitlab_status_sync.py` | GitLab sync a webhook |
| `test_api_hardening.py` | Security, error formát |
| `test_account_api.py` | Účet |
| `test_cli.py` | CLI parsování |
| `test_smoke_cleanup.py` | Úklid [SMOKE] ticketů |

Fixtures: `db` (SQLite in-memory), `fixture_data` (uživatelé, partneři, klienti).

## Frontend

Lokální build:

```bash
cd frontend && npm install && npm run build
```

Unit testy (kde existují):

```bash
cd frontend && npm test
```

CI zatím nemá samostatný frontend job — build ověřte lokálně.

## Docs-site

```bash
cd docs-site
npm install
npm run build
```

Build ověřuje TypeScript a MDX obsah.

## CI (GitHub Actions)

`.github/workflows/ci.yml`:

- trigger: `push`, `pull_request`
- job `backend-tests`: Python 3.12, `pytest`

`.github/workflows/post-deploy-smoke.yml`:

- manual + po změně smoke skriptu
