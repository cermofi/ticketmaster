Backend je **FastAPI** s prefixem **`/api`**. V produkci nginx na frontendu proxuje `/api/` na službu `api:8000`.

OpenAPI (Swagger) běží na API kontejneru na `/docs` — není vystavené přes veřejný nginx (port 3006). Pro náhled: `docker compose exec api curl -fsS http://127.0.0.1:8000/openapi.json | head`.

## Autentizace

Většina endpointů vyžaduje hlavičku:

```
Authorization: Bearer <JWT>
```

| Metoda | Endpoint | Kdo | Výsledek |
| --- | --- | --- | --- |
| `POST` | `/api/auth/login` | Partner | `{ token, user }` — e-mail + heslo |
| `POST` | `/api/auth/dev-sso` | Interní | `{ token, user }` — pouze dev, e-mail interního uživatele |
| `POST` | `/api/auth/activate` | Pozvaný uživatel | `{ token, user }` — token z invite + nové heslo |
| `GET` | `/api/auth/me` | Přihlášený | Profil aktuálního uživatele |

Login je rate-limited (`LOGIN_RATE_LIMIT_*`). Neúspěšné pokusy vrací `403`, překročení limitu `429`.

Partner se přihlašuje heslem. Interní uživatel v dev prostředí přes `dev-sso` (uživatel musí existovat v DB, `kind=internal`).

## Admin — partneři, klienti, uživatelé

Vyžaduje interní roli **Admin** nebo **DeliveryManager** (u mazání partnera jen Admin).

| Skupina | Endpointy |
| --- | --- |
| Partneři | `GET/POST /api/partners`, `DELETE /api/partners/{id}` |
| Klienti | `GET/POST /api/clients`, `PATCH/DELETE /api/clients/{id}` |
| Přiřazení klientů | `GET/POST /api/client-assignments`, `DELETE /api/client-assignments/{id}` |
| Uživatelé | `GET /api/users`, `POST /api/users/internal`, `POST /api/users/partner`, `PATCH/DELETE /api/users/{id}`, `POST .../password-reset`, `POST .../deactivate` |

Partner vidí jen své klienty a uživatele (`GET /api/users`, `GET /api/clients` filtrované podle `partner_id`).

## Chybové odpovědi

Aplikační chyby vrací JSON `{ "detail": "..." }`:

| HTTP | Význam |
| --- | --- |
| `403` | Chybí token, špatná oprávnění, neplatný login |
| `404` | Entita neexistuje |
| `422` | Validační chyba (tělo požadavku) |
| `429` | Příliš mnoho pokusů o login |

Každá odpověď obsahuje hlavičku `X-Request-ID` pro korelaci s logy API.
