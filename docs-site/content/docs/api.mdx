Backend je **FastAPI** s prefixem **`/api`**. OpenAPI specifikace: **`/api/openapi.json`**.

Interaktivní reference (Scalar) je na této stránce. V produkci jsou **`/docs/*`** i **`/api/openapi.json`** dostupné jen z interní sítě — viz [Provoz](/docs/provoz).

## Přehled

| Položka | Hodnota |
| --- | --- |
| Base URL (dev Docker) | `http://localhost:3006` |
| API prefix | `/api` |
| OpenAPI JSON | `/api/openapi.json` |
| Formát těla | `application/json` (kromě uploadu příloh) |
| Autentizace | `Authorization: Bearer <JWT>` |

Veřejné endpointy bez tokenu: `GET /api/health`, `GET /api/ready`, `GET /api/meta`.

Webhook bez JWT: `POST /api/gitlab/webhook` (autentizace přes `X-Gitlab-Token`).

## Autentizace a tokeny

Detailní popis login flow, impersonace a rate limitu: [Autentizace](/docs/autentizace).

```bash
BASE=http://localhost:3006

# Partner login
curl -sS -X POST "$BASE/api/auth/login" \
  -H 'Content-Type: application/json' \
  -d '{"email":"partner@example.test","password":"ChangeMe123!"}'

TOKEN="<jwt-z-loginu>"

curl -sS "$BASE/api/auth/me" -H "Authorization: Bearer $TOKEN"
```

## Chybové odpovědi

Unified JSON tvar:

```json
{
  "code": "permission_denied",
  "message": "Invalid e-mail or password",
  "details": null,
  "request_id": "uuid"
}
```

| HTTP | Kód | Typické příčiny |
| --- | --- | --- |
| `400` | `bad_request` / business kód | Validační chyba, nepodporovaný export |
| `403` | `permission_denied` | Chybí token, špatná role |
| `404` | `not_found` | Ticket, příloha neexistuje |
| `409` | `conflict` | Business konflikt |
| `422` | `validation_error` | Pydantic validace |
| `429` | `rate_limit_exceeded` | Příliš mnoho auth pokusů |
| `503` | `service_unavailable` | Webhook není nakonfigurován |

Response obsahuje hlavičku **`X-Request-ID`** (= `request_id` v JSON).

## Stránkování ticketů

`GET /api/tickets`:

| Parametr | Popis |
| --- | --- |
| `search`, `status`, `priority`, `type`, `resolver_team`, `partner_id`, `internal` | Filtry |
| `limit` | Default `TICKET_PAGE_DEFAULT_LIMIT` (50), max `TICKET_PAGE_MAX_LIMIT` (200) |
| `offset` | Default `0` |

Odpověď: `{ "items": [...], "total": N, "limit": L, "offset": O }`. Řazení: `created_at DESC`.

`GET/POST /api/partner-api/partners/{partner_id}/tickets` — system tickety. Přístup: Admin/DM nebo partner stejného `partner_id` (POST jen `responsible`).

## Praktické příklady

### Meta a health

```bash
curl -sS "$BASE/api/health"
curl -sS "$BASE/api/ready"
curl -sS "$BASE/api/meta"
```

### Vytvoření ticketu (partner)

Použijte hodnoty z `/api/meta`:

```bash
curl -sS -X POST "$BASE/api/tickets" \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "type": "Question",
    "priority": "Normal",
    "title": "Dotaz k integraci",
    "description": "Potřebujeme upřesnit API endpoint.",
    "client_id": null,
    "participant_ids": []
  }'
```

### Změna stavu

```bash
TICKET_ID="<uuid>"

curl -sS -X POST "$BASE/api/tickets/$TICKET_ID/transition" \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"status":"In progress"}'
```

Používejte jen stavy z `available_transitions` v detailu.

### Export Excel

```bash
curl -fsS -o export.xlsx \
  "$BASE/api/tickets/export?format=xlsx&status=New" \
  -H "Authorization: Bearer $TOKEN"
```

Detail: [Export](/docs/export).

### Upload přílohy

Povolené přípony: `.png`, `.jpg`, `.jpeg`, `.pdf`, `.txt`, `.log`, `.zip`.

```bash
curl -sS -X POST "$BASE/api/tickets/$TICKET_ID/attachments" \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/logfile.txt"
```

## Poznámky

- Swagger UI na API kontejneru je vypnuté — dokumentace je na `/docs/api`.
- Smoke check posílá `x-ticketmaster-smoke: 1` — audit se nezapisuje.
- Partner workflow používá `/api/tickets`; Partner API je pro automatizaci system ticketů.
