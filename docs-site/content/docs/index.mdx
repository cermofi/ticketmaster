TicketMaster je interní ticketovací systém pro **partnery** a **interní týmy** (L1–L3, Admin, Delivery Manager).

## Co systém dělá

- Partner vytváří a sleduje tickety svého partnera (klient, typ, priorita, komentáře, přílohy).
- Interní tým tickety přiřazuje, řeší, uzavírá a audituje.
- Tickety s týmem **L3** lze propojit s GitLab issue a synchronizovat stav oběma směry.
- Admin a Delivery Manager spravují partnery, klienty a uživatele.

## Pro koho je tato dokumentace

Pro kohokoli, kdo aplikaci **neviděl** — vývojáře, provoz, support i nové členy týmu. Každá stránka popisuje jednu oblast; začněte [Rychlým startem](/docs/spusteni).

## Stack (stručně)

| Vrstva | Technologie |
| --- | --- |
| API | FastAPI, SQLAlchemy, PostgreSQL 16 |
| UI | React 19, Vite, ASAB WebUI (hash router) |
| CLI | `ticketmaster-cli` (Python) |
| Docs | Next.js + Fumadocs (`docs-site/`) |
| Infra | Docker Compose: `db`, `api`, `frontend`, `docs`, `mailpit` |

## Kde co běží

| Prostředí | Web UI | Dokumentace | API |
| --- | --- | --- | --- |
| Lokální Docker | http://localhost:3006 | http://localhost:3006/docs/ | http://localhost:3006/api/ |
| Lokální docs dev | — | http://localhost:3007/docs | — |
| Produkce | https://ticketmaster.cermofi.cz | `/docs/` (jen interní síť) | `/api/` |

Frontend nginx proxuje `/api/` → `api:8000` a `/docs/` → `docs:3000`.
