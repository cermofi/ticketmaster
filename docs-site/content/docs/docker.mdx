## Spuštění

```bash
cp .env.example .env
docker compose up -d --build
docker compose ps
```

## Služby

### `db`

- Image: `postgres:16-alpine`
- Healthcheck: `pg_isready`
- Volume: `postgres_data`

### `api`

- Build: `backend/Dockerfile`
- Start: `ticketmaster-cli db migrate` + uvicorn (`WEB_CONCURRENCY` workerů)
- Healthcheck: `GET http://127.0.0.1:8000/api/ready`
- Volumes: `uploads`, `api_logs`
- Expose: `:8000` (jen interní síť Compose)

### `docs`

- Build: `docs-site/Dockerfile` (Next.js standalone, `basePath=/docs`)
- Healthcheck: `GET http://127.0.0.1:3000/docs`
- Expose: `:3000`

### `frontend`

- Multi-stage: Node 22 build → nginx 1.27
- Port host: **127.0.0.1:3006:80**
- Proxy: `/api/` → `api:8000`, `/docs/` → `docs:3000`
- Interní geo filtr pro `/docs` a `/api/openapi.json` (viz `frontend/nginx.conf`)

### `mailpit`

- Dev SMTP (`SMTP_HOST=mailpit`, port 1025)
- Web UI není mapovaný na host port

## Diagram

```
Host :3006 → frontend nginx
               ├─ /api/*  → api:8000
               ├─ /docs/* → docs:3000
               └─ /*      → React static
api → db:5432
```

## Proměnné prostředí

Společný blok `x-app-environment` — kompletní seznam v [Konfigurace](/docs/konfiguracni-minimum).

Poznámka: pokud `.env` chybí, Compose default pro `GITLAB_DRY_RUN` je `false`; `.env.example` má `true` pro bezpečný dev.

## Logy a restart

```bash
docker compose logs -f api frontend docs
docker compose restart api
docker compose exec api ticketmaster-cli health
```

## Rebuild po změně kódu

```bash
# jen API
docker compose build api && docker compose up -d api

# frontend + docs
docker compose build frontend docs && docker compose up -d frontend docs

# vše
docker compose up -d --build
```

## CLI uvnitř kontejneru

```bash
docker compose exec api ticketmaster-cli <příkaz>
```
